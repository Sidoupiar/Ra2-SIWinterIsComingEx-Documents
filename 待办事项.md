# 待办事项 v0.0.9

## 【已封闭】

#### 改进性功能仍然会被添加，不再接受新的独立功能。

## 可能会添加的功能

### Buff

* 触发关联 Buff。  
被挂上这种 Buff 的单位可以被触发行为激活，激活后这些单位会成为那个触发的关联对象。

### 杂项

* 限制矿车只能采某一或几种矿物。

* 适配于新经验系统的训练所和自定义渗透。

* 元素攻击。  
`[全局] -> ResistAttack=yes,yes,yes`，对免疫单位是否可以自动攻击/反击/强制攻击  
`[弹头] -> DamageType=`，决定造成的伤害类型，默认 无（需要注册表）  
`[弹头] -> Resist.Attack=`，微观设置，默认全局值  
`[单位] -> ResistType=`，决定免疫哪几种伤害类型  
`[单位] -> Resist.xxx=1`，xxx表示已有的伤害类型，对该类型伤害的受伤倍数，默认 1

* 传送弹头。  
`Warp=no`，启用逻辑  
`Warp.Force=no`，强制传送，不考虑目标位置有没有东西，no = 传送到最近的可通行地面  
`Warp.Kill=no`，强制杀死目标点的东西  
`Warp.Modes=0`，对目标的传送模式  
0 = 把目标传送到自己身边  
1 = 传送至相对坐标，以自身为原点  
2 = 传送至绝对坐标，以地编为准  
3 = 全图随机可达到的空地（包含水面）  
`Warp.Offest=0,0,0`，坐标偏移值，适配 `Warp.Modes=1,2`  
`Warp.Self=no`，把自己传送过去

## 想来想去添不上去的功能

### Buff

* Buff 的几个效果强度值判断，用 `-2147483648` 表示不判断。  
数据精度不够，暂时搞不定（而且这个数值也太怪了）。

* 记录单位类型 Buff。  
无软用。

* 弹药限制 Buff。  
当弹药超过 A 或低于 B 时均会禁用指定的武器（会尝试切换为另一个可使用的武器攻击）。  
以目前的眼光看，实现它的代价有点大，而且在功能上并非不可替代。

* 逆分摊 Buff。  
此 Buff 已被伤害转化广播 Buff 替代。

* 扩散连锁 Buff。  
此 Buff 已被 Buff 挂载监听 Buff 替代。

* 扩散弹头（抛射体）Buff。  
此 Buff 已被弹头监听 Buff 替代。

* 揭示黑幕 Buff。  
因为可以跳超武，而且这个本来也没啥用。

### 杂项

* 军团效应。  
性能堪忧，而且可以通过 Buff 简单近似模拟。

* `单位转换` 弹头的持续时间，时间结束变回去。  
使用各种 Buff 可以实现绝大多数情况的需要。

* `变更所属作战方` 弹头的持续时间，时间结束变回去。

* 添加改善 ini 编写的功能（类似于经典的块继承这种的），这类功能请提给 INI 连接器。

* 扩充文件载入范围类的功能（栗如想要支持直接读取名为 `rulesEX.ini` 的文件或者想要支持读取 `.png` 文件什么的）。

* 一看就没什么意义，甚至有些离谱的功能（栗如在游戏里打开一个界面编写 ini 代码）。

* 带有某种破坏性效果的功能（比如故意弹窗恶心别人的流氓行为）。

* 接入第三方系统，这个会有其他层面的问题，因此不做。

* 扩展底层结构，比如支持 8 人以上游戏什么的，这个是真的无能为力。
